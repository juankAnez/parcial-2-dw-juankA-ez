3. CONFIGURACIÓN DE BASES DE DATOS:
   El ambiente local debe tener:
   • MySQL corriendo en localhost:3306
   • PostgreSQL corriendo en localhost:5432
   • Credenciales configuradas en .env

4. TESTING:
   Usar la extensión "REST Client" de VS Code para probar http.http.
   NO usar curl desde terminal (menos cómodo).

5. RELACIÓN FORÁNEA:
   Es CRÍTICO que tuitions.car_id tenga FK constraint a cars.id.
   Esto previene crear tuitions con car_id que no existen.

6. FAKER.JS:
   Está instalado como dependencia dev.
   Se usa en seeders para generar datos realistas.
   Las ciudades deben ser colombianas (Bogotá, Medellín, Cali, Barranquilla, etc.)

7. EXPRESS-VALIDATOR:
   Se usa en middlewares para validar request body.
   Valida ANTES de llegar al controlador.

8. SEQUELIZE:
   Usa { alter: true } en sync para crear/actualizar tablas automáticamente.
   Sin necesidad de migraciones.

9. RESPUESTAS JSON:
   Formato estandarizado:
   {
     "status": "success|error",
     "message": "Descripción",
     "data": { ... }    // Solo si aplica
   }

10. CÓDIGOS HTTP:
    • 200: OK (GET, PUT, DELETE exitosos)
    • 201: CREATED (POST exitoso)
    • 400: Bad Request (validación falla)
    • 404: Not Found (recurso no existe)
    • 409: Conflict (violación de unicidad)
    • 500: Server Error (error del servidor)
